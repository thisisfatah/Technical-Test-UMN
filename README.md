# Aquascape Technical Test

Unity version in this project is `6000.0.59f2`. The PDF brief asked for `6000.3.12f1`, so this mismatch should be disclosed when submitting.

## What Is Implemented

- Default Windows player resolution set to `1920x1080`
- Runtime PNG import from external `RuntimeImports` folder without Unity recompile.
- File naming rules:
  - `FISH_[TYPE]_[YYYYMMDDHHMMSS].png`
  - `TRASH_[TYPE]_[YYYYMMDDHHMMSS].png`
- Fish behaviour:
  - wandering with separation and wall avoidance
  - hunger meter drain
  - seek nearest food when hunger reaches `0`
  - stop eating at `100` and wait for cooldown
  - temporary flee state when clicked
- Trash behaviour:
  - drifting float motion
  - separation and wall avoidance
  - click to clean with fade feedback
- Player interaction:
  - left click empty water to spawn food
  - left click fish to scare it
  - left click trash to remove it
- External gameplay tuning through root `config.json`
- Seed import support:
  - if `RuntimeImports` is empty, the game copies starter PNG files from `Assets/StreamingAssets/SeedImports`

## How To Run

1. Open the project in Unity `6000.0.59f2`.
2. Open [SampleScene.unity](E:/Project/Game-Project/Technical%20Test%20UMN/Assets/Scenes/SampleScene.unity).
3. Wait for script recompilation.
4. Check the hierarchy:
   - `Main Camera` is only the scene camera
   - `Aquascape Controller` has [AquascapeController.cs](E:/Project/Game-Project/Technical%20Test%20UMN/Assets/Scripts/Aquascape/AquascapeController.cs) for runtime flow, camera framing, bounds, and prefab hookups
   - under the controller there are `Environment`, `Entities`, and `Effects` roots
   - `Environment` is edited directly as normal scene objects, without any environment-builder script
5. If prefab templates do not appear automatically in [Assets/Resources/Aquascape](E:/Project/Game-Project/Technical%20Test%20UMN/Assets/Resources/Aquascape), run the Unity menu command `Aquascape > Rebuild Template Prefabs`.
6. Press Play.
7. The game creates `RuntimeImports` in the project root if it does not exist.

## How The System Works

### 1. Scene-Level Control

- [AquascapeController.cs](E:/Project/Game-Project/Technical%20Test%20UMN/Assets/Scripts/Aquascape/AquascapeController.cs) is the orchestration point for runtime systems, aquarium bounds, camera framing, and prefab references.
- Environment visuals are edited directly in the scene hierarchy under `Environment`.
- In edit mode, the controller custom inspector exposes preview buttons for layout/camera work only.
- The controller initializes:
  - world registry
  - runtime scanner
  - spawn service
  - player input
  - HUD
  - ambient bubbles
  - feedback/effects

### 2. Editable Prefab Templates

Prefab templates are intended to live in [Assets/Resources/Aquascape](E:/Project/Game-Project/Technical%20Test%20UMN/Assets/Resources/Aquascape):

- `FishEntity.prefab`
- `TrashEntity.prefab`
- `FoodEntity.prefab`

These are generated by [AquascapePrefabBootstrapper.cs](E:/Project/Game-Project/Technical%20Test%20UMN/Assets/Editor/Aquascape/AquascapePrefabBootstrapper.cs).

What you can edit in those prefabs:

- visual hierarchy
- extra child objects
- sprite renderer setup
- collider setup
- sorting order
- extra polish components you want to add later

Runtime import still injects the PNG sprite dynamically, but the object shell now comes from prefabs instead of raw `new GameObject(...)`.

### 3. Runtime Flow

1. Controller loads [config.json](E:/Project/Game-Project/Technical%20Test%20UMN/config.json).
2. Controller ensures external `RuntimeImports` exists.
3. If that folder is empty, starter PNGs from [Assets/StreamingAssets/SeedImports](E:/Project/Game-Project/Technical%20Test%20UMN/Assets/StreamingAssets/SeedImports) are copied in.
4. [RuntimeAssetScanner.cs](E:/Project/Game-Project/Technical%20Test%20UMN/Assets/Scripts/Aquascape/RuntimeAssetScanner.cs) polls the folder and validates filenames.
5. [SpawnService.cs](E:/Project/Game-Project/Technical%20Test%20UMN/Assets/Scripts/Aquascape/SpawnService.cs) loads the PNG, instantiates the correct prefab template, then injects runtime sprite/config data.
6. [FishAgent.cs](E:/Project/Game-Project/Technical%20Test%20UMN/Assets/Scripts/Aquascape/FishAgent.cs), [TrashAgent.cs](E:/Project/Game-Project/Technical%20Test%20UMN/Assets/Scripts/Aquascape/TrashAgent.cs), and [FoodItem.cs](E:/Project/Game-Project/Technical%20Test%20UMN/Assets/Scripts/Aquascape/FoodItem.cs) drive the behaviour.

## Adding New Assets At Runtime

Drop new `.png` files into the root `RuntimeImports` folder while the game is running.

Examples:

- `FISH_MACKEREL_20260401165920.png`
- `TRASH_BOTTLE_20260401165920.png`

Rules:

- only `.png` is accepted
- names outside the required convention are ignored with warnings
- new files are scanned recursively
- imports are processed once after the file size becomes stable

## Config

The external config file lives at [config.json](E:/Project/Game-Project/Technical%20Test%20UMN/config.json).

Top-level sections:

- `scanner`
- `food`
- `interaction`
- `fish`
- `trash`

`fish.types` and `trash.types` are arrays keyed by their `type` field. If a runtime filename uses a type that is not listed, the game falls back to the corresponding `default` profile.
